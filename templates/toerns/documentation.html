Website Documentation

This website is written in Django and uses a Sqlite database to store all data. Currently the site is hosted on the free hosting site pythonanywhere.com under the account andreas61667.
Please contact the webmaster Volker Petersen for any design changes or major updates to this site.

PS: The German word Törn means a trip on a ship.

Database

You can reach the database maintenance tool by going to Resource/Admin Page. All data is stored in 2 key tables:

  • The table Crew Members holds a list of all crew members that have sailed on Andreas. In addition to their name the table also holds a list of their sailing skills. The field email is current not utilized.
  • The table Toern Directory holds all information about a single trip. These are the fields in this table:
    • startDate: start of the trip using the format YYYY-MM-DD
    • endDate: end of the trip using the format YYYY-MM-DD
    • Destination: a headline description of the trip
    • Geo Region: the geographical region of the trip. The website currently supports these regions: Atlantic, Caribbean, Great Lakes, Pacific
    • Maptable: the name of an Sqlite table with the waypoints along the trip. This table is used to show the journey on a Google Map.
    • Boat: the name and description of the boat. This field is also being used to add information like:
      a pdf file with a trip log (using a hyperlink pointing to the file surrounded by the keywords openLink and closeLink)
      a link to another website with tracker infor (e.g. Yellowbrick) or race results (e.g. Yacht Scoring)
      a linefeed the break the text into multiple lines (using keyword linefeed)
      Please see example implementations in the existing Toern Directory entries.
    • Miles: the total miles sailed on this trip
    • Days At Sea: the number of days at sea
    • Skipper: the skipper for this trip
    • Crew: a list of crew members on this trip. You can select the crew members from a list of all sailors in the table Crew Members. If a sailor is not yet in the Crew Member table they must be added first in this table before they can be added to a trip.
    • Image: the name and location of an image you want to show for this trip. All images should be stored in the folder static/images. An entry in this field should then look like: images/nameOfTripImage.jpg
    • Picture Link: a link to another website hold a picture album. I usually add a link to my Google Photo Albums
  • Please don't make any changes to the tables Users and Groups found on the Admin page.

Process to add a trip to this website

For every trip that you want to add to this website you should have these components:

  1. A picture to be featured on the website. The picture should be landscape with an aspect ratio of 800x600 (W x H). A resolution of 100 pixels/inch is sufficient. Preferred image formats are webp, png, or jpg.
    The tool Upload Content can be found at Resources/Admin.
  2. The trip stats. Start date, end date, total miles sailed, and number of days at sea.
  3. An SQL file with the route sailed on this trip. The tool Upload Content found on the Resources/Admin page can be used to convert a GPX route file (can be exported from Expedition) into am SQL file. This tool will also automatically upload the converted file into this wensite's database.
  4. Ideally a Google Photos album with pictures and videos from the trip.

First step is to update the Crew table with any crew members that have not previously sailed on Andreas.
Second step is to upload all the content to the website (see above).
The third and final step is to update the Toern Directory table with all the trip stats, picture name, and Google Photo Album link.
The Database tables can be edited on the Resources/Admin page.
